## B3 — Metric Card Redesign

### Verified type (lib/deliverable/templates.ts)
```ts
type StatSlot = {
  kind: "stat";
  id: string;
  label: string;
  value: string;
  source_url?: string;
  source_label?: string;
  freshness_token?: string;  // optional in slot, required as "" in metric item
};
```

### Verified metric item schema (lib/project/items.ts)
Fields required: `kind: "metric"`, `report_id`, `label`, `value`, `freshness_token` (required — use `""` when slot lacks one). Optional: `source_url`, `source_label`, `metric_slug`.

### New files
- `app/p/[id]/StatCard.tsx` — `"use client"` component  
- `lib/briefcase/metric-item.ts` — `buildMetricItem(slot, deliverableId)` helper (parallel to `qa-item.ts`)

### Card layout (value-first, left accent, collapsible source)
```
┌─│ $485,000         ← text-2xl font-bold
  │ MEDIAN LIST PRICE ← text-[10px] uppercase tracking-widest text-gray-400
  │ ▸ Source          ← <details><summary> — collapsed by default
  │ + File            ← only when briefcase.projectId non-null
```
Left border: `borderLeftWidth: 3px`, color `var(--chart-accent, #00d4aa)` (already injected by brand theme on page).

### Filing flow
`StatCard` calls `useBriefcase().fileItem(buildMetricItem(slot, deliverableId))` — same pattern as `BriefcaseChat.fileAnswer`. Shows "Filed ✓" for 1800ms then resets. Button hidden when no active project.

### Change in page.tsx
Replace `renderStat(st)` map with `<StatCard key={st.id} slot={st} deliverableId={id} />`. The `id` is available via closure (page component scope).

## B5 — Buttons That Lie

### BTN-4: Mobile hamburger (`components/landing/Header.tsx`)
Convert to client component (`"use client"`), add `useState(false)` for `mobileOpen`. Wire hamburger `onClick`. Add mobile nav drawer below the bar (same links/auth as desktop, hidden `md:hidden`).

### BTN-2: "Open in mail" (`app/p/[id]/DeliveryButtons.tsx`)
Replace `buildMailto()` with `buildGmailLink()` — same subject/body construction, different base URL: `https://mail.google.com/mail/?view=cm&fs=1&su=…&body=…`. Change label to "Open in Gmail". Single intentional target, no multi-client ambiguity. `to=` left blank (sender fills recipient).

### BTN-3/10: Print-hide for bridged dock (`components/briefcase/AiBriefcasePill.tsx`)
Wrap `{open && bridged && <AskAiDock …/>}` in `<div className="print-hide">`. The non-bridged panel is already inside the `print-hide` div at line 40.

### BTN-5: TemplateSwitcher for owners only (`app/p/[id]/page.tsx`)
After fetching the deliverable (line 412+), get authenticated user via `createClient(await cookies())`. `isOwner = user?.id === data.user_id`. `DeliverableRow.user_id` is already in the fetched row. Render `<TemplateSwitcher>` only when `isOwner`. Adds one auth call; no new queries (user_id is on the deliverable row).

### BTN-6: Remove upload stub (`components/highlighter/AskAiDock.tsx`)
Delete the `<div className="mt-2 flex justify-start">…Upload your data · soon…</div>` block entirely.

### BTN-7: Wire "Request this data" (`components/highlighter/HighlightPopup.tsx`)
Change `onClick={onClose}` to: POST `{ action: "data_request", detail: question }` to `/api/meter` (existing endpoint, no new route), then call `onClose()`. The `question` prop (current search text) is available in scope.

### BTN-8/9: MCPInstall Cursor/Windsurf (`components/landing/MCPInstall.tsx`)
Remove `cursor` and `windsurf` entries from the tab map. Add a one-line note below the Claude tab: "Cursor and Windsurf support is on the roadmap — HTTP MCP transport arriving soon."
