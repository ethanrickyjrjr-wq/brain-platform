## Mid-session redirect worth recording

The PR #19 secrets normalize merged before Vercel's env vars were renamed, which silently broke prod signup for a window. User caught it from the prod side, not from CI. Recovery was two PRs (`f738ed4` canonical read, `c3b9d0a` add legacy fallback) inside ~45 minutes. The right pattern when shipping a rename across a code/deploy boundary is **ship the fallback FIRST, then rename**, not "rename code, then rename deploy config" — even when the two PRs are minutes apart, the gap is enough to break prod.
